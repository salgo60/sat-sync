Roundtripping mellan SAT-kartan, OpenStreetMap och Wikidata

Denna sida sammanfattar utmaningar och frågeställningar kring synkronisering ("roundtripping") mellan tre öppna datakällor:

  • SAT-kartan
  • OpenStreetMap (OSM)
  • Wikidata

Målet är att förbättra datakvaliteten utan att skapa dubbla objekt eller förlora information när data uppdateras i flera system.

Övergripande mål

  • Identifiera skillnader mellan systemen.
  • Avgöra vilken källa som är mest korrekt.
  • Förenkla återkoppling av förbättringar.
  • Minska manuellt arbete.
  • Bygga spårbara arbetsflöden.

OSM Notes som arbetsflöde

OSM Notes kan användas för att rapportera observationer utan att direkt ändra kartan. Det gör det möjligt för lokala OSM-mappare att verifiera informationen innan ändringar görs.

Exempel:

  • Fel namn
  • Saknat objekt
  • Objekt verkar ha flyttats
  • Behöver lokal kontroll
  • Behöver verifieras mot officiell källa

Möjliga arbetsflöden

1. SAT → OSM

  1. Identifiera diff.
  2. Verifiera.
  3. Skapa OSM Note eller direkt ändring.
  4. När ändringen är godkänd uppdateras SAT.

2. OSM → SAT

  1. Nya objekt upptäcks.
  2. Importeras efter kvalitetskontroll.

3. Wikidata → OSM

  1. Lägg till saknade wikidata-taggar.
  2. Undvik felaktiga kopplingar.

Tekniska utmaningar

  • Stabila identifierare.
  • Objekt som flyttas.
  • Objekt som delas upp eller slås samman.
  • Versionshantering.
  • Historik.
  • Automatiska kontra manuella ändringar.
  • Kvalitetssäkring innan synkronisering.

Frågeställningar

  • Hur identifieras samma objekt i tre olika system?
  • När ska en diff automatiskt accepteras?
  • När krävs mänsklig granskning?
  • Kan OSM Notes användas som gemensam arbetskö?
  • Bör SAT lagra OSM-id och Wikidata-id permanent?
  • Hur hanteras objekt som saknar Wikidata?
  • Hur undviks att samma fel rapporteras flera gånger?
  • Hur prioriteras vilka differenser som ska åtgärdas först?
  • Kan diff-rapporten generera färdiga arbetslistor?
  • Hur mäts datakvaliteten över tid?

Vision

Ett långsiktigt mål är ett öppet arbetsflöde där SAT, OpenStreetMap och Wikidata förstärker varandra istället för att utvecklas separat. Diff-rapporter används för att hitta avvikelser, OSM Notes används för samarbete, och förbättringar återförs till samtliga system på ett spårbart sätt.
